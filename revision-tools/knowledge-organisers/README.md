# Knowledge Organisers — One Page Per Topic

A **knowledge organiser** condenses an entire topic onto ~1–2 sides of A4: the must-know facts, definitions, methods and exam reminders, in tight tables and bullet lists. They're built for **last-minute review** and **quick daily recall** — not first learning (use the full topic files for that).

> How to use: read the organiser, then **cover it and write out** the key tables/definitions from memory. Print them, stick them up, quiz yourself. Pair with the matching [flashcard deck](../flashcards/README.md) and [mini-paper](../mini-papers/README.md).

## Where they fit in your revision
1. **Learn** the topic from the full [topic file](../../README.md#-resource-map).
2. **Drill recall** with the [flashcards](../flashcards/README.md).
3. **Condense & review** with the organiser here — your go-to for the final weeks and exam morning.
4. **Prove it** under time with the [mini-paper](../mini-papers/README.md) and full [mock papers](../mock-papers/README.md).

Each organiser ends with a **"Top exam reminders"** box — the common mistakes and discriminators for that topic. Reading those eight boxes the night before an exam is some of the highest-value revision you can do.
