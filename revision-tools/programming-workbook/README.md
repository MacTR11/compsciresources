# Programming Workbook

Hands-on coding challenges — the best preparation for **Component 02** (writing/tracing code) and the **NEA programming project**. Solutions are in **Python 3** (you may use any language), with OCR pseudocode shown where it helps, and each challenge is graded ★ Foundation / ★★ Developing / ★★★ Stretch.

## Contents
| Workbook | Covers |
|----------|--------|
| [1 — Core programming skills](01-core-programming-skills.md) | Sequence/selection/iteration, strings, lists & 2D lists, functions & procedures, scope, value vs reference, recursion, validation & error handling, file handling, OOP & inheritance |
| [2 — Algorithms & data structures](02-algorithms-and-data-structures.md) | Linear/binary search, bubble/insertion/merge/quick sort, stack, queue (incl. circular), linked list, BST + traversals, and a Dijkstra stretch — each with its Big O |

## How to use
1. **Attempt first**, in a real editor — actually run your code.
2. Reveal the solution only to check, then note what you missed.
3. Re-do anything you couldn't complete unaided a few days later.

> Why this matters: Paper 2 asks you to *write and trace* algorithms, and the NEA is 20% of the grade for building a working program. Reading about code is not the same as writing it — this is where those marks are won. See also the [pseudocode guide](../guides/pseudocode-guide.md) and the [NEA guide](../../component-03-04-programming-project/README.md).
